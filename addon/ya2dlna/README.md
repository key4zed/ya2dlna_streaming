# Аддон Ya2DLNA Streaming

Этот аддон запускает приложение ya2dlna_streaming как сервис внутри Home Assistant. Он предоставляет два основных сервиса:

- **REST API** (порт 8000) – для управления потоком и настройками из интеграции Home Assistant.
- **DLNA-сервер** (порт 8001) – для трансляции аудио на DLNA-рендереры.

## Конфигурация

Настройте аддон через интерфейс Home Assistant (Настройки → Аддоны → Ya2DLNA Streaming → Конфигурация). Доступны следующие параметры:

- **ya_music_token** – Токен API Яндекс Музыки (необязательный).
- **x_token** – X‑Token для авторизации Яндекс.Станции.
- **cookie** – Cookie Яндекс (альтернатива x_token).
- **ruark_pin** – PIN-код для Ruark R5 (если используется Ruark).
- **local_server_host** – Хост для сервера стриминга (по умолчанию: `0.0.0.0`).
- **local_server_port_dlna** – Порт DLNA-сервера (по умолчанию: `8001`).
- **local_server_port_api** – Порт REST API (по умолчанию: `8000`).
- **stream_quality** – Качество аудио (`192`, `320` и т.д.).
- **debug** – Включить отладочное логирование.
- **mute_yandex_station** – Отключать звук на Яндекс Станции во время трансляции (по умолчанию: `true`).

Если `ya_music_token` не указан, аддон попытается получить его автоматически из `x_token` или `cookie`.

## Использование

После запуска аддона REST API будет доступен по адресу `http://<hassio_ip>:8000`. Кастомный компонент `ya2dlna` взаимодействует с этим API для управления потоком.

### Управление настройками через API

Настройки можно изменять динамически через эндпоинты REST API:

- `GET /settings` – получить текущие настройки.
- `POST /settings` – обновить настройки (тело запроса в JSON).

Интеграция Home Assistant предоставляет сервисы `ya2dlna.update_settings` и `ya2dlna.get_settings` для удобного управления.

### Потоковое вещание

- `POST /ha/stream/start` – начать трансляцию с выбранной Яндекс.Станции на выбранный DLNA-рендерер.
- `POST /ha/stream/stop` – остановить трансляцию.
- `GET /ha/stream/status` – получить статус текущего потока.

## Логи

Логи доступны в панели логов аддона. При включённой опции `debug` выводятся подробные отладочные сообщения.
