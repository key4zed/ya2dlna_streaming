# Project Debug Rules (Non-Obvious Only)

- Логирование на русском языке, сообщения могут содержать кириллицу. Для фильтрации используйте `grep` с учётом кодировки UTF-8.
- Основные логи выводятся в stdout/stderr. В Docker контейнерах логи можно просмотреть через `docker logs <container>`.
- Для отладки стриминга включите `APP_DEBUG=true` в `.env`. Это увеличит детализацию логов.
- Если стриминг не запускается, проверьте наличие токена Яндекс.Музыки (`APP_YA_MUSIC_TOKEN`) и корректность PIN Ruark (`APP_RUARK_PIN`), если используется.
- Ошибка "ModuleNotFoundError: No module named 'core'" указывает на неправильный `PYTHONPATH`. Убедитесь, что переменная окружения `PYTHONPATH` установлена в `/app/src` (в Dockerfile) или что вы запускаете скрипт из корневой директории проекта.
- При использовании Home Assistant addon ошибка "s6-overlay-suexec: fatal: can only run as pid 1" означает, что не установлены `init: false` и `S6_OVERLAY_SUEXEC=0`. Проверьте config.yaml и Dockerfile addon.
- DLNA устройство может не обнаруживаться, если UPnP отключён в сети. Убедитесь, что устройство доступно по multicast DNS (mDNS). Можно использовать `upnpclient` для ручного поиска.
- Если Яндекс.Станция не отвечает, проверьте, что она в той же сети и что mDNS работает. Используйте `avahi-browse` для поиска устройств.
- В тестах используются моки `unittest.mock.MagicMock`. Фикстуры в `conftest.py` предоставляют предварительно настроенные зависимости. Для запуска одного теста используйте `pytest tests/test_client.py -v`.
- При отладке асинхронного кода используйте `asyncio.run()` или запускайте через `pytest` с плагином `pytest-asyncio`. Не забывайте про `await` в корутинах.
